# learning-client

React SPA — the UI layer. Holds no business logic and talks only to the Node BFF (`learning-server`), never directly to `learning-core-api`.

## Architecture

```
React (learning-client, :5000)   ← this repo
    │  HTTP (REST) + WebSocket
    ▼
Node BFF (learning-server, :5001)
    │  HTTP
    ▼
.NET Core API (learning-core-api, :5002)
    │
    ▼
PostgreSQL
```

- **Auth**: `AuthContext` (`src/context/AuthContext.tsx`) calls `POST {VITE_API_URL}/api/users/auth` on the BFF, expects `{ status, message, user }`, and stores the full authenticated user (`id`, `username`, `email`, `roles`) in `localStorage` under `auth_user` — available app-wide via `useAuth()`. `ProtectedRoute` (`src/components/ProtectedRoute.tsx`) gates routes on `isAuthenticated`.
- **Realtime**: `WebSocketMessageComponent` and `SocketNotificationComponent` connect to `{VITE_SERVER_URL}/chat` and `{VITE_SERVER_URL}/notifications` on the BFF via the shared `useWebSocket` hook.
- **Routing/shell**: `main.tsx` defines public routes (`/`, `/login`, `/logoff`) and protected routes wrapped in `ProtectedRoute` + `AppShell` (`/dashboard`, `/users`, `/cartpage`, plus a few `/test*` demo routes). `AppShell.tsx` composes `Sidebar`, `BottomNav`, and the routed page.

## Tech stack

| Concern | Choice |
|---|---|
| Framework | React 19, React Router 7 |

| Typed API client | Generated from the BFF's OpenAPI spec via `openapi-typescript` (see below) |
| Lint | ESLint 10 + `typescript-eslint` |

## API contract: typed calls to the BFF

`learning-server` exposes an OpenAPI spec (generated by `tsoa` from its controllers, written to `learning-server/spec/swagger.json`). This client turns that into TypeScript types:

```bash
npm run generate:types
```

This runs `openapi-typescript ../learning-server/spec/swagger.json -o src/types/bff-api.generated.ts`. Hand-written convenience aliases (`UserDto`, `RoleDto`, `LoginSuccessResponse`, `ErrorResponse`) live in `src/types/bff-api.ts` and re-export from the generated file — import from there, not the generated file directly.

Requires `learning-server`'s spec to exist; run `npm run tsoa:gen` in `learning-server` first (or `npm run build`, which does it automatically) if the spec is missing or stale.

## Environment variables

| Var | Purpose | Dev default |
|---|---|---|
| `VITE_API_URL` | BFF REST base URL | `http://localhost:5001` |
| `VITE_SERVER_URL` | BFF WebSocket base URL | `ws://localhost:5001` |

Set in `.env.development` / `.env.production`.

## How to run

```bash
npm install
npm run dev
```

Dev server runs on port `5000` (see `vite.config.ts`). Requires `learning-server` running on `5001` (which in turn requires `learning-core-api` on `5002`).

## Production build

```bash
npm run build
```

Build output goes to `dist/`, deployed by Azure Static Web Apps.

## Lint

```bash
npm run lint
```
